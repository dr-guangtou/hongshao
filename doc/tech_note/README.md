# Technical notes — the HongShao emulators

Journal-style documentation of the project's two emulators of the
halo-to-stellar-profile relation for massive central galaxies
($M_{\rm peak} > 10^{13} M_\odot$, TNG300, $z = 0.4$–$2.0$). The intended
reader understands the motivation — predicting massive-galaxy stellar mass
profiles and their evolution from halo assembly histories — but has not
followed the experiments; every model ingredient is introduced from
scratch, with equations and pedagogical figures.

| note | model | status |
|---|---|---|
| [1 — The statistical emulator](01_statistical_emulator.md) | heteroscedastic conditional Gaussian on $[\mathrm{DiffMAH}(4), c_{200c}]$, PCA profile modes, the multi-epoch AR(1) block-pinned product | **adopted** — the accuracy and generative product |
| [2 — The transport kernel (1ch-mof)](02_transport_kernel.md) | lognormal efficiency window, power-law-tail deposits, migration clock; official $z \le 1.5$ fit scope; the 1-D empirical stochastic layer | **adopted** — the physics companion |
| [3 — The two-channel alternative (2ch-exp)](03_two_channel_alternative.md) | compact Gaussian + wide exponential channel with a halo-dependent split | documented alternative — never adopted |

Suggested reading order: 1 → 2 → 3. Note 3 depends on note 2's machinery;
notes 1 and 2 are self-contained.

## The adopted product stack, in one line each

- **Predict or draw masses/profiles** → the statistical emulator
  (`hongshao/emulator.py`, `profile_emulator.py`, `multi_epoch.py`; manual
  in `doc/emulator_manual.md`).
- **Reason about assembly physics** → the 1ch-mof kernel at the official
  $z \le 1.5$ scope
  (`experiments/exp40_epoch_objective/outputs/latestart.npz[theta_z15]`),
  with the $z=2.0$ column of every QA table read as the extrapolation
  diagnostic of the dissipative era.
- **Draw populations from the kernel** → add the 1-D empirical stochastic
  layer (`experiments/exp41_stochastic_layer/stage2_draws.py`,
  `draw_deltas` + `drawn_cogs`; deviation pool in its
  `outputs/stage1_dist.npz`).

## The standardized QA figure sets

The pedagogical figures in these notes are *not* the QA record. The most
up-to-date standardized QA sets (the `hongshao.qa` layout: kpc and $R_e$
aperture/annulus masses truth-vs-model per epoch, halo-mass-binned
residuals, the 2-D observational planes with energy/floor scores, the
best/worst case gallery) live with their experiments — gitignored, so
regenerate on a fresh checkout:

| model | figure set | regenerate with |
|---|---|---|
| statistical emulator (block product) | `experiments/exp37_multi_epoch/figures/qa_*_exp37_block.*` | `experiments/exp37_multi_epoch/run.py qa` |
| adopted kernel + stochastic layer (official $z \le 1.5$ scope, mean + draws) | `experiments/exp41_stochastic_layer/figures/qa_*_exp41_kernel_layer.*` | `experiments/exp41_stochastic_layer/stage2_draws.py report` |
| 2ch-exp and 1ch-mof at the five-epoch scope (like-for-like) | `experiments/exp38_deposit_rethink/figures/qa_*_exp38_{2ch-exp,1ch-mof}.*` | `experiments/exp38_deposit_rethink/stage2_multiepoch.py report` |

(All commands from the repo root, prefixed `PYTHONPATH=. uv run python`,
with `HONGSHAO_DATA_DIR` exported.)

## Figures

All figures (and the two GIF animations) are generated by the scripts in
[`scripts/`](scripts/):

```bash
export HONGSHAO_DATA_DIR=/path/to/tng300_mah_mprof
PYTHONPATH=. uv run python doc/tech_note/scripts/make_figures_note1.py
PYTHONPATH=. uv run python doc/tech_note/scripts/make_figures_note2.py
PYTHONPATH=. uv run python doc/tech_note/scripts/make_figures_note3.py
```

`common.py` holds the shared helpers, including a deliberately
self-contained mirror of the kernel forward model (validated by its
`__main__` self-check) so the illustrations run from an analytic DiffMAH
history without loading the experiment harness. Requirements beyond the
repo: the processed catalog `data/processed/tng300_072_z0p4.fits` (note 1)
and the experiment output files named above (notes 2–3).

## PDF export

```bash
bash doc/tech_note/scripts/build_pdf.sh
```

renders each note plus a combined volume with a table of contents to
`doc/tech_note/pdf/` (gitignored — derived artifacts; requires pandoc and
xelatex). GIF animations are replaced by a representative static frame in
the PDFs.
