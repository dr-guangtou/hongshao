# Session Handover — 2026-06-22

## Goal
Build the "Ultimate SHMR": a portable, probabilistic model that predicts a
massive central galaxy's stellar-mass profile from its halo's mass assembly
history (MAH). This session took it from "evaluation metrics" through a working,
portable, calibrated emulator. **Next session: try symbolic regression (PySR) to
find a better closed-form functional model for the emulator mean.**

## Completed This Session (exp07 → exp11, all merged to master + pushed)
- **exp07 — evaluation suite** (`hongshao/metrics.py`: CRPS, log-score, PIT,
  interval calibration, AIC/BIC; `tng_data.cog_sigma_dex`). Score predictors in
  aperture/annulus masses (not per-radius CoG dex — 93% correlated). Use
  CRPS+calibration, not RMS. Residual scatter is correlated across apertures
  (must draw correlated scatter). AIC/BIC must be decorrelated (annulus space).
- **5-kpc CoG fit default** (`COG_FIT_RMIN_KPC=5.0`): the inner residual S-shape
  is an unresolved <5 kpc effect; cached `rdm_rms` median 0.0060→0.0013 dex.
- **exp08 — first probabilistic emulator**: conditional multivariate Gaussian
  P(aperture masses | M0, MAH-PCA(4)), linear mean + full residual covariance.
  Predict the observable directly (A) ≫ predict rdm_* params then reconstruct (B);
  B fails because the radial-DiffMAH shape params are a degenerate *coordinate
  system* that buries a real (~24% of at-fixed-M0 variance) concentration signal —
  NOT because the shape signal is weak (corrected an earlier overstatement).
- **exp09 — ceiling check**: a gradient-boosted-trees ceiling does NOT beat the
  linear model (CRPS 0.0855 vs 0.0851; shuffle control collapses to M0-only). The
  relation is essentially linear → keep the closed-form equation. Best-fit
  coefficients: SHMR slope dlogM*/dlogM0 steepens outward 0.55→1.05; assembly
  dependence grows outward via MAH-PC2 (timing).
- **exp10 — our own DiffMAH fits** (`hongshao/diffmah.py`): rolling power law,
  k=3.5, fit over t≥2 Gyr (early history unresolved; follows Hearin+2021 t>1 Gyr).
  Median fit RMS 0.063 dex. The 4 portable DiffMAH params carry ~88% of the
  MAH-PCA(4) assembly signal. As a *compressor* DiffMAH is worse than PCA (≈PCA-2)
  but an equal *predictor* and far better *portable* feature.
- **exp11 — portable emulator** (THE milestone): cached DiffMAH params in the
  dataset (`dmah_*`, `DMAH_FIT_TMIN_GYR=2.0`); rebuilt the exp08 emulator on them.
  Matches the MAH-PCA version under the exp07 suite: identical calibration
  (54/72/91/95), full covariance +1.43 nats, residual correlation 0.52, CRPS
  0.0882 vs 0.0849. Probabilistic painting works. → **portable, calibrated,
  full-covariance Ultimate-SHMR emulator.**

## Problems / Blockers
- None open. Outer aperture bins (e.g. 100–150 kpc) carry more scatter (low SB,
  projection, ICL) — a data limit, not a framework limit (checked, see below).

## Key Decisions
- Emulator = four per-aperture **linear mean** models + one joint Gaussian with a
  **full covariance** (correlated scatter), on **portable DiffMAH features**.
- The mean is linear (exp09 ceiling check); features are portable (exp11).
- **Robustness settled (checked, no separate experiment):** perturbing aperture
  boundaries by a few kpc changes mean CRPS by 0.0002 dex; adding a 100–150 kpc
  bin works (inner bins unchanged, still calibrated, new bin CRPS 0.104). The
  emulator is definition-agnostic → can match a survey's apertures and refit.
- Goal is an observation-constrainable framework → prefer closed-form parametric
  models, not black-box ML (trees/GP only as an achievability ceiling).
