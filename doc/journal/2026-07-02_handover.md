# Session Handover — 2026-07-02 (exp29 multi-epoch: puff model, consistency limit, honest eval)

## Goal
Phenomenological forward-model emulator: halo MAH (+ halo-only secondaries) →
stellar-mass CoG at 5 epochs (z=0.4/0.7/1.0/1.5/2.0). This session pushed the
multi-epoch modeling to its limit and overhauled the evaluation methodology.

## What this branch did (`exp29-puff-model`, merged to master)
Building on the prior finding (deposit *shape* is fine per-epoch; the problem is
multi-epoch *consistency*):
- **Puff-up model** (`puff_fit.py`): one consistent history, mass frozen, widths
  migrate post-deposition (ratio & diffusion laws). **Underperforms** the loose
  z-dependent fit (7% vs 4.5% on the old footing) — width migration is a weaker lever
  than mass-distribution flexibility; diffusion law nearly inert.
- **Radial comparison** (`model_compare.py`): all compact models fail at the
  intermediate-radius core→envelope transition, worst for BCGs.
- **Free-mass NNLS floor** (`nnls_floor.py`): the decisive result — free per-deposit
  mass does NOT relieve the tension. Each epoch fits ALONE to ~0.2%, but one shared
  mass vector (a single consistent additive history) caps the JOINT fit at ~12%. **The
  binding limit is the single consistent additive Gaussian-sum history itself**, not
  the parameterization. Reaching the ceiling needs a non-additive (transport) primitive.
- **Real MAH** (`real_mah_test.py`): `dipfree_mah` used the SMOOTH DiffMAH fit, erasing
  merger bursts. Re-tested with the real de-dipped `peak_history` (running-max: keeps
  bursts, removes dips, ~60 gappy deposits). Fits get worse (smooth curve flattered the
  model); the real MAH is honest AND carries the merger events an event-driven width
  model would need.
- **Corrected evaluation** (`integrated_check.py`): dropped the inner-3-kpc mask (high-z
  Re<3 kpc → inner holds >50% of the mass; masking flattered the metric). All radii.
- **Standard mass QA** (`mass_qa.py`): reusable `evaluate()`, two bin sets (kpc +
  R_half), aperture M*(<R) + envelope M*(>R), two figure types (truth-vs-model values;
  truth-vs-relerr). Run parallel to the profile metric after every fit.
- **Honest final scorecard** (`final_scorecard.py`): all 4 models, real MAH, all radii.

## Honest numbers (real MAH, all radii, n=45), profile max|rel| epoch-avg
- independent (ceiling): **1.9%**
- loose-quad (best consistent-ish): **9.9%**
- puff-ratio: **10.9%**
- free-mass NNLS floor: **18.5%** (worst in max|rel| — L2 objective spikes the worst radius)

Integrated masses: cumulative apertures M*(<10..<100 kpc) ~0.01 dex for ALL models;
relative outskirt M*(>2Re) ~0.02 dex for the good ones; ONLY the fixed-kpc far outskirt
at high z fails (loose −0.31 dex M*(>50 kpc) at z=2) — because 50 kpc is ~15 R_half out
for compact high-z galaxies (R_half 12.7 kpc z=0.4 → 3.3 kpc z=2), i.e. the negligible
far tail, benign in R_half units. So the emulator is far better on integrated /
size-relative quantities than the ~10% profile number implies.

## Key decisions / methodology (now standard)
- **Evaluate over ALL radii** for multi-epoch (never inner-mask; high-z mass is inner).
- **Two metrics in parallel**: profile max|rel| AND `mass_qa.evaluate()` (kpc + R_half
  aperture/envelope masses). A "floor"/"ceiling" must state its metric (L2 ≠ max|rel|).
- **Use the real de-dipped MAH** (`peak_history`) for honest numbers — but note the model
  scripts still DEFAULT to the smooth `dipfree_mah` (see open item).
- **Always surface new figures** (path + content) every figure-generating turn — now an
  explicit AGENTS.md rule.
